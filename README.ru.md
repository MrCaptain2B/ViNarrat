# ViNarrat

Социальные сцены для Foundry VTT — портреты, диалог, фон, всё в реальном времени.  
Игроки видят то, что ты строишь, в тот же момент. Никаких токенов, никакой подготовки.

> **v0.7.8** — работает на Foundry v13 и v14

---

## Что это?

ViNarrat — полноэкранная сцена, где ты перетаскиваешь портреты, меняешь фон, пишешь диалог и выбираешь, кто говорит. Всё в реальном времени, всё видно игрокам сразу после нажатия Broadcast.

Это не движок визуальных новелл. Это **режиссёр сцены**. Каждый NPC-портрет, на которого игроки обычно тупо смотрят, теперь стоит на сцене, меняет эмоции, и всегда понятно, кто говорит.

---

## Быстрый старт (для ведущего)

```
/vn           Открыть сцену
/vnreq текст  Отправить запрос ведущему (игроки)
/vnportrait   Открыть свой портрет (игроки)
/vnrejoin     Пересинхронизироваться после обновления
```

**Впервые:**
1. Напиши `/vn`.
2. Панель **Portraits** → **Import from Actors**. Выбери папку с актёрами.
3. Открой **Locations** → добавь фоны для локаций.
4. Перетащи портрет на сцену. Нажми на него — шкала, отражение, блокировка.
5. Напиши диалог в Scene Control. Выбери говорящего в панели слева.
6. Нажми **Broadcast**. Теперь игроки видят то же, что и ты.

## Что можно делать

### 🎭 Портреты на сцене
Перетащи портрет из панели на сцену. Двигай, меняй размер, отрази, заблокируй. Несколько эмоций на портрет — игроки могут переключать их сами, ведущий видит сразу.

Панель спикеров слева показывает все портреты с цветными полосками: синий = игрок, красный = NPC. Нажми на имя — назначь говорящим. Активный спикер подсвечен золотом.

### 💬 Диалог
Написал текст, выбрал кто говорит — готово. Одно окно или два столбца. У диалога есть ползунки прозрачности, ширины, высоты, размера шрифта и выравнивания. Можно кликнуть по тексту прямо на сцене и редактировать inline.

### 🖼 Пресеты (сохранения сцены)
Сохрани всю сцену — фон, яркость, все портреты с позициями/масштабом/эмоциями, говорящего, настройки диалога. Назови, загрузи потом. Экспортни в `.json`, импортни на другом мире.

### 📡 Трансляция и приглашения
Трансляция отправляет всё — фон, портреты, диалог, спикера, цвета темы — всем игрокам. Приглашение работает точечно: кнопка Invite, выбираешь игрока. Работает через сокет Foundry.

### 👁 Игроки
У игроков лёгкая панель: портреты (могут редактировать свой портрет — имя, титул, эмоции), кнопка Attention (пинг ведущему с пульсацией), и реджоин через макрос.

### 🔐 Права доступа

| Настройка | По умолчанию | Что даёт |
|-----------|-------------|----------|
| `permManage` | GM | Полное управление сценой |
| `permBroadcast` | GM | Трансляция |
| `permApproveClaims` | GM | Подтверждение Attention |
| `permAddRequests` | Player | Отправка запросов ведущему |

## Реджоин (для игроков)

Когда трансляция активна, а ты закрыл сцену (или потерял соединение), придёт виспер с кликабельной **@UUID** ссылкой на макрос. Нажми — переподключишься.

Или создай макрос сам:

```
game.freevisualnovel.rejoin()
```

Перетащи на хотбар. Жми когда надо синхронизироваться.

---

## API

```js
game.freevisualnovel.open()                // Открыть сцену
game.freevisualnovel.close()               // Закрыть
game.freevisualnovel.setBackground(path)
game.freevisualnovel.addPortrait(data)
game.freevisualnovel.addRequest(text)
game.freevisualnovel.clearStage()
game.freevisualnovel.importActorPortraits()
game.freevisualnovel.rejoin()              // Переподключиться к трансляции
game.freevisualnovel.getBroadcastState()   // Проверить активность трансляции
```

## Честно о багах

- Это **бета**. Основное работает (трансляция, портреты, регионы), но миграции при смене формата данных не будет.
- Импортированные пресеты ссылаются на пути изображений с твоего сервера. Если тех же картинок нет на целевом мире — они не покажутся. Копирования файлов при импорте не происходит.
- Цвета темы теперь транслируются игрокам, но RGB акцента считается автоматически из hex-пикера. Если выглядит не так — обнови страницу.
- FilePicker (для выбора картинок) использует стандартную реализацию Foundry v13+. Нативный импорт пресетов использует обычный `<input type="file">`, потому что FilePicker в v13/14 придирчив к расширениям.
- Никаких зависимостей. Ни JSZip, ничего. Работает с любой системой.
