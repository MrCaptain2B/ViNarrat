# Free Visual Novel — Прогресс

## Суть проекта
Модуль для Foundry VTT — полноэкранная оверлейная VN-сцена. GM управляет сценой (фон, портреты, диалог, броадкаст), игроки видят её в реальном времени, могут управлять своими портретами (эмоции, кнопка Attention/Claim).

## Структура файлов
- `scripts/visualnovel.js` — вся логика (класс приложения, панели, драг, сокеты, настройки)
- `templates/visualnovel.hbs` — Handlebars-шаблон всей VN
- `style/visualnovel.css` — вся стилизация
- `module.json` — манифест модуля (с `"socket": true`)

## Ключевые возможности (сделано)

### Сцена
- Полноэкранный оверлей, `z-index: 9999`, поверх Foundry UI
- Фон — изображение или градиент, регулировка яркости (Scene Settings)
- Смена эмоций портрета — cross-fade через overlay `<img>` (0.4s)
- Перетаскивание портретов мышью (только GM, `role < 3` — ранний возврат)
- Выделение портрета кликом (только GM)

### Панели (открываются из тулбара)
- **Locations** — список + фильтр (search, tag, group) + пагинация (30/кнопка Show More) + добавление/редактирование
- **Portraits (GM)** — то же + импорт из актёров + добавление на сцену + удаление
- **Portraits (Player)** — открывается по кнопке "My Portrait" в сцен-контролях / по `/vnportrait` / по кнопке в тулбаре VN; видит ТОЛЬКО свои портреты (`userId` match); может редактировать (имя, заголовок, теги, изображения, эмоции); кнопки удаления/импорта/добавления на сцену скрыты
- **Scene Settings** — яркость, theme colors (bg + accent), диалог (ширина/высота/прозрачность/выравнивание/текст/шрифт), спикер бокс (шрифт)
- **Presets** — сохранение/загрузка снепшотов сцены

### Диалог
- Блок внизу по центру, ширина/высота/прозрачность/выравнивание/размер шрифта — настраиваются
- Спикер индикатор — всегда виден когда есть спикер (авто-размер, только шрифт настраивается)
- Спикер бар — выбор говорящего (только GM, `role >= 3`)

### Attention / Claim
- Игрок нажимает ✋ на своём портрете → сокет `{type: "claim"}` → GM видит пульсирующую рамку + кнопку ✓
- GM нажимает ✓ → портрет становится спикером
- Визуал: claim = затемнение + drop-shadow + медленная пульсация на wrapper (2s, цвет акцента)
- Обычный портрет (не спикер, не claim) = только dim (`brightness(0.55)`)

### Player Portrait Control
- Опционально (вкл/выкл в Configure Module — `playablePortraits`)
- Привязка портрета к игроку через `userId` в форме + авто-тег `player:Name`
- Emotion strip показывается на своих портретах (даже без выделения)
- Emotion change от игрока → сокет `{type: "emotion"}` → GM обновляет и броадкастит всем
- Emotion strip над портретом (absolute), макс 5 эмоций

### Броадкаст
- GM нажимает Play → сокет `{type: "state"}` со всеми портретами, спикером, claimed, inviteMode
- **inviteMode**: "All Online" (все) или "Stage Only" (только у кого портрет на сцене)
- Stop → сокет `{type: "stop"}` → все игроки закрывают VN; `_claimed` очищается
- GM не применяет свой же броадкаст (`role >= 3` ранний возврат в `_applyVNState`)
- Только GM может броадкастить (`_broadcastVNState` проверяет `role < 3`)

### Разное
- Theme colors (bg + accent) хранятся И в `vndata` (быстрое сохранение), И в module settings (Configure Module)
- DOM-based фильтрация для search/tag, context-based для group/pagination
- Ограничение: макс 10 портретов на сцене
- `game.freevisualnovel` API: `open()`, `close()`, `setBackground()`, `addPortrait()`, `addRequest()`, `clearStage()`, `importActorPortraits()`
- Чат команды: `/vnreq <text>`, `/vnportrait` / `/vnedit`

## Статус тестирования
- **Броадкаст GM ↔ игрок** — ✅ Работает
- **Emotion change от игрока** — ✅ Работает
- **Игрок не может двигать/выделять чужие портреты** — ✅ Работает (может выделить только свой, менять эмоции, Attention)
- **Attention/Approve (player ✋ → GM ✓)** — ❌ Не работает. Нужно:
  - Когда GM выбирает портрет из списка (spiker bar) и у этого портрета active attention — портрет становится спикером
  - При этом пульсирующая обводка снимается
  - Обводка НЕ должна сбрасываться у всех игроков при выборе одного — только у того, кого назначили спикером

3. **Селектор вкладок Foundry** — кнопка с иконками чата/актёров/сцен, клик открывает поп-аут (не закрывая VN)
4. **Погода/время/температура** — интеграция с Simple Calendar, цветовой градиент
5. **CSS-анимации** — fade/slide для показа/скрытия элементов вместо бинарного show/hide
6. **Сохранение сессии броадкаста** — сейчас если GM или игрок нажал ×, обратно зайти в тот же диалог нельзя. Нужно: пока active broadcast — сессия жива, удалять сцену только когда все вышли (GM + все игроки)
